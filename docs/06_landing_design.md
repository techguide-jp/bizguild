# ランディングページ・全体改善設計書（B2B向け）

## 1. サービスポジショニング

### BizGuildとは

**起業家・事業者コミュニティ向けのプラットフォーム提供サービス**

- 管理者が自分のネットワークを「ビジネスギルド」として運営できる
- メンバー間の協業・相互紹介を仕組み化
- 管理者はコミュニティの価値向上と収益化が可能

### ビジネスモデル

```
┌─────────────────────────────────────────────────────────┐
│  BizGuild（プラットフォーム提供）                         │
│    ↓ 契約・導入                                          │
│  管理者（イベンター、営業、コミュニティ主催者）             │
│    ↓ 招待・審査                                          │
│  メンバー（フリーランス、個人事業主、小規模事業者）         │
└─────────────────────────────────────────────────────────┘
```

## 2. ターゲット分析（※LP上には表示しない・内部資料）

### プライマリターゲット

| タイプ                 | 特徴                                                     | 課題                                     |
| ---------------------- | -------------------------------------------------------- | ---------------------------------------- |
| **イベンター**         | 起業家イベントを主催、参加者との継続的な関係構築が難しい | イベント後のつながりを維持・活性化したい |
| **営業・紹介業**       | 起業家ネットワークが広い、紹介で収益を得ている           | 紹介の管理が煩雑、成果追跡ができない     |
| **コミュニティ主催者** | オンラインサロンやSlackグループ運営                      | メンバー同士のビジネス連携を促進したい   |
| **協会・団体運営者**   | 士業や専門家の協会を運営                                 | メンバー間の協業機会を創出したい         |

### ターゲットの共通ニーズ（※訴求の軸として活用）

1. **ネットワークの資産化**: 人脈を継続的な価値に変えたい
2. **コミュニティ運営の効率化**: 手作業での紹介・マッチングをシステム化
3. **収益化の仕組み**: 紹介手数料やメンバーシップで収益を得たい
4. **差別化**: 他のコミュニティとの差別化要素が欲しい

## 3. LP設計方針

### 目的

**「自分のネットワークでこれを使いたい」と思わせる**

### 設計原則

1. **「ターゲットはあなた」とは言わない** - 課題と解決策で自然に共感させる
2. **押し付けない** - 機能紹介ではなく「実現できること」を見せる
3. **想像させる** - 自分のネットワークで使うイメージを持たせる
4. **行動を促す** - 興味を持ったら次のステップ（問い合わせ）へ

### LP構成の流れ

```
共感（課題）→ 可能性（解決）→ 具体化（機能）→ 安心（料金・導入）→ 行動（CTA）
```

## 4. 価値提案（Value Proposition）

### メインメッセージ

**「あなたのネットワークを、ビジネスが生まれる場所に」**

### サブメッセージ

人脈は持っている。でも、それを活かしきれていない。
BizGuildは、つながりを「ビジネスが生まれるコミュニティ」に変えるプラットフォームです。

### 3つの価値（※「管理者向け」とは言わない）

#### 1. つながりを資産に変える

> 名刺交換で終わっていた関係が、継続的なビジネス関係に

- メンバーのスキル・サービスが可視化される
- 誰が何を得意としているかが一目でわかる
- 必要なときに、必要な人にすぐつながれる

#### 2. 紹介が自然に生まれる仕組み

> 「いい人いない？」を仕組み化

- メンバー同士がお互いのサービスを紹介
- 紹介の成果が見える化される
- 紹介した人にもメリットが還元される

#### 3. 質の高いネットワークを維持

> 信頼できる人だけが集まる安心感

- 招待制・審査制でメンバーの質を担保
- 安心して協業・紹介ができる環境
- コミュニティの価値が継続的に高まる

## 5. 収益モデル（導入者の売上ルート）

### 5.1 基本方針：まずユーザーを増やす

**最初から会費を取ると、ユーザーが増えにくい。**

推奨アプローチ:

1. **初期**: メンバー無料 → ユーザー数を増やす
2. **成長期**: 成果報酬型の収益を得る（紹介手数料など）
3. **成熟期**: 必要に応じて会費導入を検討

### 5.2 紹介手数料の仕組み

#### 基本設計：成約金額の10%（成果報酬）

```
紹介が成約した場合のみ、成約金額の10%が手数料として発生

┌─────────────────────────────────────────────────────────┐
│  成約金額: 100,000円                                     │
│                                                          │
│  手数料 10% = 10,000円                                   │
│    └─ 紹介者へ: 7,000円（紹介のお礼）                    │
│    └─ 運営者へ: 3,000円（プラットフォーム利用料）         │
│                                                          │
│  サービス提供者の受取: 90,000円                          │
└─────────────────────────────────────────────────────────┘
```

#### 利用者にとって「負担 < メリット」になる理由

| 比較対象                 | コスト                            | BizGuild紹介手数料          |
| ------------------------ | --------------------------------- | --------------------------- |
| 広告費（リスティング等） | クリック単価100-500円、成約率1-3% | **成約時のみ10%**           |
| 紹介営業代行             | 成約金額の20-30%                  | **成約金額の10%**           |
| 人材紹介                 | 年収の30-35%                      | -                           |
| クラウドソーシング手数料 | 5-20%                             | **10%（かつ紹介者に還元）** |

#### 競合サービスの手数料比較

| サービス                 | 手数料体系          | 備考                                       |
| ------------------------ | ------------------- | ------------------------------------------ |
| **クラウドソーシング系** |                     |                                            |
| ランサーズ               | 16.5%（税込）       | 10万円以下の部分。10万超は10%、100万超は5% |
| クラウドワークス         | 20%（税込）         | 10万円以下。10-20万は10%、20万超は5%       |
| ココナラ                 | 22%（税込）         | 一律。出品者負担                           |
| **マッチング系**         |                     |                                            |
| ビザスク                 | 30%                 | スポットコンサル。謝礼から差し引き         |
| Yenta                    | 月額6,800円〜       | サブスク型。成約手数料なし                 |
| YOUTRUST                 | 月額費用 + 決定報酬 | 採用時は想定年収の20-35%                   |
| **紹介・営業代行系**     |                     |                                            |
| Saleshub                 | 成約報酬の50%       | アポ報酬 + 成約報酬の合計から              |
| 紹介営業代行会社         | 20-30%              | 成約金額に対して                           |
| 顧問紹介サービス         | 30-50%              | 顧問料から継続的に徴収                     |
| **人材紹介系**           |                     |                                            |
| 一般的な人材紹介         | 年収の30-35%        | 採用決定時に一括                           |
| ハイクラス転職           | 年収の35-40%        | エグゼクティブ向け                         |
| **BizGuild**             | **10%**             | **成約時のみ。うち7%は紹介者へ還元**       |

#### BizGuildの優位性

| ポイント           | 他サービス                       | BizGuild                     |
| ------------------ | -------------------------------- | ---------------------------- |
| 手数料率           | 16-35%                           | **10%**                      |
| 成約前コスト       | 月額費用・広告費が必要           | **成約まで0円**              |
| 紹介者へのメリット | なし（プラットフォームが総取り） | **7%を紹介者に還元**         |
| 関係性             | プラットフォーム経由の初対面     | **信頼できるコミュニティ内** |
| 継続性             | 毎回手数料発生                   | **直接取引への移行も可**     |

**ポイント**:

1. **成約しなければ0円** - リスクなし
2. **広告費より安い** - 成約単価で比較すると圧倒的に低コスト
3. **紹介者がいるから成約率が高い** - 信頼がある状態でスタート

#### 手数料率のオプション（運営者が設定可能）

| 設定 | 手数料率 | 紹介者 | 運営者 | 想定用途                 |
| ---- | -------- | ------ | ------ | ------------------------ |
| 低め | 5%       | 3%     | 2%     | 高単価サービス向け       |
| 標準 | 10%      | 7%     | 3%     | 一般的なサービス         |
| 高め | 15%      | 10%    | 5%     | 紹介を強く促進したい場合 |

※ 運営者はコミュニティの性質に合わせて設定可能

### 5.4 収益シミュレーション（会費なしモデル）

※ 紹介手数料は標準設定（成約金額の10%、うち運営者3%）で計算

#### ケース1: 立ち上げ期（30名）

```
【想定】
・月間紹介成約: 3件 × 平均10万円 = 30万円

【運営者収入】
・紹介手数料（3%）: 30万円 × 3% = 9,000円/月
────────────────────────────────
月間収入合計: 9,000円

【支出】
・BizGuild利用料: 0円（フリープラン）
────────────────────────────────
月間利益: 9,000円

※ この段階は「収益化」より「ネットワーク構築」がメイン
※ 紹介者は 30万円 × 7% = 21,000円/月 を獲得
```

#### ケース2: 成長期（100名）

```
【想定】
・月間紹介成約: 15件 × 平均15万円 = 225万円
・月間協業案件: 3件 × 平均30万円 = 90万円

【運営者収入】
・紹介手数料（3%）: 225万円 × 3% = 67,500円/月
・案件手数料（5%）: 90万円 × 5% = 45,000円/月
・プレミアム枠: 10,000円 × 3名 = 30,000円/月
────────────────────────────────
月間収入合計: 142,500円

【支出】
・BizGuild利用料: 50,000円（固定費プラン）
────────────────────────────────
月間利益: 92,500円
年間利益: 1,110,000円
```

#### ケース3: 成熟期（200名）

```
【想定】
・月間紹介成約: 40件 × 平均20万円 = 800万円
・月間協業案件: 8件 × 平均50万円 = 400万円

【運営者収入】
・紹介手数料（3%）: 800万円 × 3% = 240,000円/月
・案件手数料（5%）: 400万円 × 5% = 200,000円/月
・プレミアム枠: 10,000円 × 10名 = 100,000円/月
────────────────────────────────
月間収入合計: 540,000円

【支出】
・BizGuild利用料: 50,000円（固定費プラン）
────────────────────────────────
月間利益: 490,000円
年間利益: 5,880,000円

※ 紹介者全体では 800万円 × 7% = 56万円/月 を獲得
※ メンバーにとっても「紹介すると収入になる」インセンティブ
```

### 5.5 利用者（メンバー）にとってのメリット

| 立場             | 手数料            | メリット                                         |
| ---------------- | ----------------- | ------------------------------------------------ |
| **紹介される側** | 成約の10%を支払い | 広告費ゼロで新規顧客獲得。成約しなければ費用ゼロ |
| **紹介する側**   | なし              | 成約の7%を受け取れる。紹介するだけで収入に       |
| **運営者**       | なし              | 成約の3%を受け取れる。コミュニティが活性化       |

**結論**:

- 紹介される側は「広告費の代わり」と考えれば圧倒的に安い
- 紹介する側は「紹介するだけで報酬」がもらえる
- 全員にメリットがある設計

## 7. ページ構成

### 7.1 2つのターゲットへのアプローチ

#### 課題

- **導入者（契約者）**: BizGuildを導入してコミュニティを運営したい人
- **利用者（メンバー候補）**: コミュニティに参加してサービスを使いたい人

この2者は目的が異なるため、同じLPでは訴求しにくい。

#### 結論: ページを分ける

| ページ               | 対象               | 目的                   | URL         |
| -------------------- | ------------------ | ---------------------- | ----------- |
| **導入者向けLP**     | コミュニティ運営者 | 契約・問い合わせ獲得   | `/`         |
| **メンバー先行登録** | 参加希望者         | 興味ある人のリスト収集 | `/waitlist` |

#### 理由

1. **訴求ポイントが違う**
   - 導入者: 収益化、運営効率化、ネットワークの資産化
   - メンバー: 協業機会、紹介を受けられる、質の高いネットワーク

2. **CTAが違う**
   - 導入者: 「話を聞いてみる」→ 商談
   - メンバー: 「先行登録する」→ リスト獲得

3. **導入者の安心材料になる**
   - 「すでにメンバー希望者が○○名います」
   - 先行登録数を導入者への営業材料に使える

#### ページ間の導線

```
導入者向けLP（/）
    │
    ├─→ 「話を聞いてみる」→ 問い合わせ
    │
    └─→ 「メンバーとして参加したい方はこちら」
            │
            ↓
        メンバー先行登録（/waitlist）
            │
            └─→ 「自分でコミュニティを立ち上げたい方はこちら」
                    │
                    ↓
                導入者向けLP（/）
```

## 8. コピーライティング案

### 設計方針

各セクションで視点を変え、重複を避ける：

- **ヒーロー（What）**: 何が実現できるか
- **課題共感（Why）**: なぜ今のままではダメなのか
- **解決策（How）**: どうやって解決するのか

---

### ヒーロー（What：何が実現できるか）

- **メイン**: あなたのネットワークを、ビジネスが生まれる場所に
- **サブ**: イベント参加者、ビジネス仲間、専門家のつながり。BizGuildは、そのネットワークを「協業・紹介が自然に生まれるコミュニティ」に変えます。

---

### 課題共感（Why：なぜ今のままではダメなのか）

**セクションタイトル**: こんな経験、ありませんか？

1. **せっかくのつながりが活かせていない**
   「名刺は増えるのに、ビジネスにはつながらない」

   → イベントや交流会で知り合っても、その後のフォローが難しく、関係が途切れてしまう

2. **「誰かいい人いない？」に応えきれない**
   「紹介したいのに、誰を紹介すればいいかわからない」

   → 知り合いは多いのに、それぞれの専門や強みを把握しきれていない

3. **紹介しても、その後が見えない**
   「紹介して終わり、で本当によかったのか」

   → 紹介した相手がどうなったか追えず、次の紹介に活かせない

---

### 解決策（How：どうやって解決するのか）

**セクションタイトル**: BizGuildなら、こう変わります

1. **全員のスキル・サービスを一覧で把握**
   メンバー登録時に専門分野やサービス内容を入力。誰が何を得意としているか、検索ですぐに見つかる。

   → 「この案件なら○○さんに相談しよう」が即座に判断できる

2. **紹介の仕組みをシステム化**
   「おすすめ」機能でメンバー同士を紹介。紹介が成約につながれば、紹介者にも報酬が還元される。

   → 紹介する側にもメリットがあるから、自然と紹介が生まれる

3. **招待制・審査制で質を担保**
   誰でも参加できるオープンな場ではなく、管理者が招待・審査したメンバーだけが参加。

   → 信頼できる人同士だから、安心して協業・紹介ができる

---

### CTA

- **比較検討向け**: 機能を詳しく見る → `/features`
- **導入検討向け**: 話を聞いてみる → `/contact`
- **補足コピー**: あなたのネットワークで、何が実現できるか。一緒に考えてみませんか？

## 10. よくある質問（FAQ）

### 10.1 コミュニティオーナー向け

#### 導入について

**Q. 既存のSlackやDiscordコミュニティから移行できますか？**
A. はい、既存のコミュニティメンバーを招待する形で移行できます。BizGuildはコミュニケーションの置き換えではなく、「ビジネス機能の追加」として併用することも可能です。

**Q. メンバーが10名未満でも始められますか？**
A. はい、少人数からでも始められます。むしろ、小規模から始めて運用を固めてから拡大することを推奨しています。

**Q. どのくらいの規模のコミュニティに向いていますか？**
A. 10名〜200名程度のコミュニティを想定しています。メンバー同士が「顔と名前が一致する」規模感が、紹介や協業が生まれやすい環境です。

#### 運営について

**Q. 運営にどのくらいの時間がかかりますか？**
A. 初期設定後は、メンバー審査（1件あたり5分程度）と、月次のデータ確認（30分程度）が主な作業です。紹介や協業はメンバー間で自然に発生します。

**Q. メンバーからの会費は取れますか？**
A. はい、設定可能です。ただし、まずは無料でメンバーを集め、コミュニティが活性化してから会費を検討することを推奨しています。

**Q. 紹介手数料の割合は変更できますか？**
A. はい、コミュニティの性質に合わせて5%〜15%の範囲で設定可能です。

#### 費用について

**Q. 開発パートナー契約後、追加費用は発生しますか？**
A. 開発期間中の追加費用はありません。正式リリース後は月額2.5万円（永久50%OFF）のみです。

**Q. 途中で解約できますか？**
A. 開発パートナー契約は、開発完了までの協力が前提となります。正式リリース後は、1ヶ月前の通知で解約可能です。

---

### 10.2 メンバー向け

#### 参加について

**Q. 誰でも参加できますか？**
A. いいえ、各コミュニティのオーナーによる審査があります。これにより、信頼できるメンバーだけで構成されたネットワークが維持されます。

**Q. 参加費用はかかりますか？**
A. コミュニティによって異なります。多くのコミュニティは参加無料で運営されており、紹介が成約した場合のみ手数料が発生する成果報酬型です。

**Q. 複数のコミュニティに参加できますか？**
A. はい、複数のコミュニティに同時に参加できます。それぞれのコミュニティで異なるビジネス機会が生まれます。

#### 利用について

**Q. 自分のサービスを宣伝してもいいですか？**
A. はい、プロフィールにサービス内容を登録できます。ただし、過度な営業活動はコミュニティのルールで制限される場合があります。

**Q. 紹介料はどうやって受け取れますか？**
A. 紹介が成約し、報酬が確定した後、登録した銀行口座に振り込まれます（月末締め翌月払い）。

**Q. 紹介した案件がうまくいかなかった場合は？**
A. 紹介料は成約時にのみ発生します。紹介後に成約しなかった場合、費用は一切発生しません。

#### 安心・信頼について

**Q. 個人情報は安全ですか？**
A. はい、プロフィール情報は同じコミュニティのメンバーのみに公開されます。外部には公開されません。

**Q. トラブルが起きた場合はどうすればいいですか？**
A. まずはコミュニティのオーナーにご相談ください。BizGuildとしてもサポート体制を整えています。

## 14. 競合・差別化（※内部資料）

### 類似サービス

| サービス                      | 特徴                     | BizGuildとの違い                       |
| ----------------------------- | ------------------------ | -------------------------------------- |
| オンラインサロン（DMMなど）   | コンテンツ配信中心       | 協業・紹介に特化                       |
| ビジネスマッチング（Yenta等） | 1対1のマッチング         | コミュニティとして運営できる           |
| Slack/Discord                 | コミュニケーションツール | ビジネス機能（商品・紹介・案件）が内蔵 |

### 差別化ポイント

1. **協業・紹介に特化**: 単なるコミュニケーションではなくビジネス成果に直結
2. **コミュニティ運営が収益につながる**: 紹介手数料の仕組み
3. **審査制の仕組み化**: 質の高いネットワークを維持できる
