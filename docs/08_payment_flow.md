# 決済・支払いフロー設計書

## 1. 検討が必要な課題

### 1.1 カード決済手数料の扱い

- **現状の問題**: 手数料率（10%など）に決済手数料が含まれていない
- **課題**: 決済手数料（Stripe等で約3.6%）を誰が負担するかが不明確

### 1.2 支払いフローの明確化

- **現状の問題**: ユーザー → BizGuild → サービス提供者への流れが不明確
- **課題**: 着金タイミングと支払いタイミングが不明

---

## 2. カード決済手数料の扱い案

### 案1: 決済手数料をサービス提供者が負担

```
成約金額: 100万円
├─ 決済手数料（3.6%）: -3.6万円（サービス提供者負担）
├─ 手数料（10%）: -10万円
│   ├─ 紹介者: 7万円
│   ├─ 運営者: 3.5万円（固定費プランの場合）
│   └─ BizGuild: 0.5万円
└─ サービス提供者の受取: 86.4万円
```

**メリット**:

- シンプルで分かりやすい
- 運営者・紹介者の取り分が明確

**デメリット**:

- サービス提供者の負担が大きい
- 成約金額が大きいほど負担が増える

---

### 案2: 決済手数料を全員で按分

```
成約金額: 100万円
├─ 決済手数料（3.6%）: -3.6万円
├─ 実質成約金額: 96.4万円
├─ 手数料（10%）: -9.64万円（実質成約金額ベース）
│   ├─ 紹介者: 6.75万円
│   ├─ 運営者: 3.37万円
│   └─ BizGuild: 0.32万円
└─ サービス提供者の受取: 86.76万円
```

**メリット**:

- 負担が分散される
- 公平性が高い

**デメリット**:

- 計算が複雑
- 手数料率が実質的に変動する
- 説明が難しい

---

### 案3: 決済手数料を手数料率に含める（推奨）

```
成約金額: 100万円
├─ 手数料（13.6%）: -13.6万円
│   ├─ 決済手数料（3.6%）: 3.6万円（BizGuildが負担）
│   └─ プラットフォーム手数料（10%）: 10万円
│       ├─ 紹介者: 7万円
│       ├─ 運営者: 3.5万円
│       └─ BizGuild: -0.5万円（決済手数料3.6万円 - プラットフォーム手数料0.5万円 = 実質3.1万円負担）
└─ サービス提供者の受取: 86.4万円
```

**メリット**:

- サービス提供者への負担が明確
- 手数料率が分かりやすい
- 説明しやすい

**デメリット**:

- BizGuildの実質負担が大きい（決済手数料を負担するため）

---

### 案4: 決済手数料を別途請求（月額に含める）

```
成約金額: 100万円
├─ 手数料（10%）: -10万円
│   ├─ 紹介者: 7万円
│   ├─ 運営者: 3.5万円
│   └─ BizGuild: 0.5万円
└─ サービス提供者の受取: 90万円

※ 決済手数料（3.6%）は月額費用に含まれる
```

**メリット**:

- 手数料率がシンプル
- 成約ごとの計算が簡単

**デメリット**:

- 月額費用が高くなる可能性
- 成約が少ない場合でも決済手数料を負担することになる

## 3. 支払いフローの明確化

### 3.1 推奨フロー

```
【決済・分配フロー】

1. ユーザーがBizGuild経由で支払い
   └─ 成約金額: 100万円

2. BizGuildが決済代行サービス（Stripe等）で受け取り
   └─ 着金: 100万円（決済手数料3.6%は自動差引）

3. BizGuildが手数料を分配
   ├─ 紹介者: 7万円（即時または月末締め翌月払い）
   ├─ 運営者: 3.5万円（即時または月末締め翌月払い）
   └─ BizGuild: 0.5万円

4. サービス提供者への支払い
   └─ 86.4万円（成約金額 - 手数料 - 決済手数料）
   └─ タイミング: 即時 or 月末締め翌月払い
```

### 3.2 支払いタイミングの選択肢

#### 案A: 即時払い

- 成約確定後、即座に各関係者へ支払い
- **メリット**:
  - 資金繰りが早い
  - 信頼性が高い
- **デメリット**:
  - 返金対応が複雑
  - 資金管理が大変

#### 案B: 月末締め翌月払い（推奨）

- 月末に締めて、翌月に支払い
- **メリット**:
  - 資金管理が容易
  - 返金対応がしやすい
  - 一般的なビジネス慣行に合致
- **デメリット**:
  - 支払いまで時間がかかる（最大1ヶ月）

#### 案C: 週次払い

- 毎週決済をまとめて支払い
- **メリット**:
  - バランスが取れている
  - 資金繰りが比較的早い
- **デメリット**:
  - 管理がやや複雑

## 4. 料金体系への反映案

### 4.1 推奨: 案3（決済手数料を手数料率に含める）+ 案B（月末締め翌月払い）

```
【固定費プラン】
- 初期費用: 60万円
- 月額: 1万円
- 手数料: 成約金額の10%（決済手数料3.6%は別途BizGuildが負担）
  ├─ 紹介者: 7%
  ├─ 運営者: 3.5%
  └─ BizGuild: 0.5%（実質は決済手数料3.6%を負担）

【支払いタイミング】
- 月末締め翌月払い
- 例: 1月の成約 → 2月末に支払い
```

### 4.2 表示の改善案

手数料の流れセクションに以下を追加：

```
成約金額: 100万円
├─ 決済手数料（3.6%）: 3.6万円（BizGuild負担）
├─ プラットフォーム手数料（10%）: 10万円
│   ├─ 紹介者: 7万円
│   ├─ 運営者: 3.5万円
│   └─ BizGuild: 0.5万円
└─ サービス提供者の受取: 86.4万円

※ 支払いタイミング: 月末締め翌月払い
```

## 5. 実装時の考慮事項

### 5.1 返金対応

- 返金時の手数料分配の取り扱い
- 既に支払い済みの紹介者・運営者への返金請求
- 返金手数料の負担者

### 5.2 税務

- 各関係者への支払い時の源泉徴収の有無
- 確定申告の対応
- インボイス制度への対応

### 5.3 資金管理

- BizGuildが一時的に資金を預かる期間の管理
- 資金の安全性（信託口座等の検討）
- 資金の流れの透明性

### 5.4 透明性

- 各関係者が支払い状況を確認できるダッシュボード
- 決済履歴の表示
- 手数料の内訳表示

### 5.5 決済代行サービスの選定

- Stripe、Square、PayPal等の比較
- 手数料率の確認
- 日本国内での利用可否
- APIの使いやすさ
